# GitHub Updater — Developer Guide

## Requirements

- [Node.js](https://nodejs.org/) v18 or higher
- [Git](https://git-scm.com/)

## Setup

```bash
git clone https://github.com/Alowster/github-updater.git
cd github-updater
npm install
npm start
```

Use `npm run dev` to start with DevTools open.

## Project structure

```
github-updater/
├── src/
│   ├── main.js          # Electron main process (IPC, GitHub API, auto-update logic)
│   ├── preload.js       # Secure renderer ↔ main bridge
│   └── renderer/
│       └── index.html   # UI (vanilla JS, no framework)
├── assets/
│   ├── icon.png         # App icon (256×256)
│   ├── icon.ico         # Windows icon
│   └── tray-icon.png    # System tray icon (16×16 or 32×32)
├── apps-config/
│   └── example-apps.json
└── package.json
```

## Build

```bash
npm run build         # Windows portable (.exe)
npm run build:local   # Windows, output to /dist without packaging
npm run build:linux   # Linux (.AppImage)
```

Built files are placed in `dist/`.

## Release

Releases are published via the GitHub Actions workflow at `.github/workflows/release.yml`.
Trigger it manually from **Actions → Release → Run workflow**.

The workflow:
1. Checks out the repo on `windows-latest`
2. Runs `npm ci` and `npm run build`
3. Uploads the `.exe` as a build artifact

To publish directly to GitHub Releases, use `npm run release` (requires a `GH_TOKEN` secret with `contents: write` permission).

## Self-update

The app can update itself. Set the target repo in `src/main.js`:

```js
const UPDATER_REPO = 'Alowster/github-updater'; // already set
```

Attach the compiled `.exe` as an asset to each GitHub Release and the app will detect and offer the update automatically.

## Icons

Place in `assets/`:

- `icon.png` — 256×256px, transparent background
- `icon.ico` — Windows icon (convert from PNG at [icoconvert.com](https://icoconvert.com))
- `tray-icon.png` — 16×16 or 32×32px

## GitHub Release asset format

For auto-download to work, releases must have binary assets attached:

```
Release v2.0.0
├── my-app-v2.0.0.exe      ← Windows executable
├── my-app-v2.0.0.zip      ← Portable version
└── Source code (zip)      ← auto-generated by GitHub
```

If no assets are attached, the updater opens the release page in the browser instead.

## Dependencies

| Package | Purpose |
|---|---|
| `electron` | Desktop app shell |
| `electron-builder` | Packaging and publishing |
| `electron-store` | Persistent settings and app config |
| `node-fetch` | GitHub API requests |

## Version format

The app compares versions using the GitHub Release tag. Supported formats:

- `v1.0.0` ✅
- `v2.3.1` ✅
- `1.0.0` ✅ (prefix `v` is optional)
